ABSTRAK

Implementasi Factory Pattern & Chain of Responsibility pada Workflow Persetujuan Pengadaan Barang/Jasa. Penelitian ini bertujuan membangun sistem approval pengadaan berjenjang yang adaptif terhadap nilai transaksi menggunakan pendekatan design pattern. Berdasarkan studi awal pada 3 perusahaan skala menengah, ditemukan keluhan utama berupa ketidakjelasan jenjang approval (100% responden) dan minimnya riwayat persetujuan (100%). Sistem yang diusulkan menerapkan Factory Pattern guna membuat workflow handler secara terpusat serta Chain of Responsibility untuk memproses persetujuan sesuai nominal (≤50 jt, 51–250 jt, >250 jt).

Implementasi menggunakan Spring Boot (Java 11) dengan tiga role: Manager, Kepala Divisi, dan Direksi. Setiap approval mencatat riwayat lengkap (role, remark, timestamp). REST API disediakan untuk antarmuka web (HTML/CSS/JS). Hasil user testing terhadap 3 responden menunjukkan tingkat keberhasilan 88.9% setelah iterasi perbaikan pada tombol approve dan tampilan status. Sistem telah dideploy di AWS EC2 (t2.micro) dan berjalan stabil. Penelitian ini membuktikan bahwa kombinain Factory Pattern dan Chain of Responsibility menghasilkan workflow approval yang terstruktur, fleksibel, dan mudah dipelihara.

Kata Kunci: Factory Pattern, Chain of Responsibility, workflow approval, pengadaan berjenjang, audit trail, Spring Boot.

4.3 Metode Program Java (Factory + Chain of Responsibility)

ApprovalHandlerFactory.java

public static ApprovalHandler createWorkflow() {
  ManagerHandler m = new ManagerHandler();
  DivisionHeadHandler d = new DivisionHeadHandler();
  DirectorHandler dir = new DirectorHandler();
  m.setNext(d); d.setNext(dir);
  return m;
}

ManagerHandler.java (meneruskan ke next)

if (request.getStatus() == WAITING_MANAGER) {
  request.addHistory("MANAGER", "Disetujui Manager");
  request.setStatus(request.nextStatusAfter(WAITING_MANAGER));
} else if (next != null) next.handle(request);

BAB 5 – PENJELASAN METODE PROGRAM JAVA

Setiap class memiliki tanggung jawab spesifik: ProcurementRequest menyimpan state dan aturan nextStatusAfter(), ApprovalService mengelola penyimpanan in-memory dan panggilan ke workflow factory. ApprovalController menyediakan REST endpoints. Chain of Responsibility memastikan bahwa setiap handler hanya bertindak jika status sesuai, lalu dilanjutkan ke handler berikutnya bila ada.

BAB 6 – PROTOTYPE ANTARMUKA & USER TESTING

User testing terhadap 3 responden (Manager, Kepala Divisi, Staff Procurement) menyatakan bahwa workflow approval terstruktur dan riwayat approval otomatis membantu proses audit. Tingkat keberhasilan mencapai 88,9% setelah perbaikan tombol dan tampilan status berdasarkan masukan iterasi.

BAB 7 – FULL SCRIPT PROGRAM JAVA (kode inti)

Kode lengkap (ApprovalStatus, ProcurementRequest, semua handler, service, controller) tersedia di repository source code yang dapat diunduh pada bagian akhir laporan ini. Cuplikan kode strategis di atas menggambarkan implementasi design pattern utama.

BAB 8 – DEPLOYMENT & OPERASI (AWS EC2)

Deployment dilakukan pada Amazon Linux 2023 (t2.micro). Proses menjalankan JAR dengan java -jar procurement-approval-factory-1.0.jar --server.port=8081 & disown. Security Group mengizinkan port 8081. Saat ini aplikasi berjalan stabil dengan background process.

BAB 9 – KESIMPULAN & REKOMENDASI

Kesimpulan: Factory Pattern berhasil memusatkan pembuatan rantai handler, Chain of Responsibility memberikan fleksibilitas approval manual per role, history tercatat lengkap, dan deployment cloud berhasil dilakukan. Rekomendasi prioritas: mengganti penyimpanan in-memory dengan PostgreSQL (P0), menambah notifikasi email (P1), dan dashboard analitik (P2).
