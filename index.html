ABSTRAK

Implementasi Factory Pattern & Chain of Responsibility pada Workflow Persetujuan Pengadaan Barang/Jasa. Penelitian ini bertujuan membangun sistem approval pengadaan berjenjang yang adaptif terhadap nilai transaksi menggunakan pendekatan design pattern. Berdasarkan studi awal pada 3 perusahaan skala menengah, ditemukan keluhan utama berupa ketidakjelasan jenjang approval (100% responden) dan minimnya riwayat persetujuan (100%). Sistem yang diusulkan menerapkan Factory Pattern guna membuat workflow handler secara terpusat serta Chain of Responsibility untuk memproses persetujuan sesuai nominal (≤50 jt, 51–250 jt, >250 jt).

Implementasi menggunakan Spring Boot (Java 11) dengan tiga role: Manager, Kepala Divisi, dan Direksi. Setiap approval mencatat riwayat lengkap (role, remark, timestamp). REST API disediakan untuk antarmuka web (HTML/CSS/JS). Hasil user testing terhadap 3 responden menunjukkan tingkat keberhasilan 88.9% setelah iterasi perbaikan pada tombol approve dan tampilan status. Sistem telah dideploy di AWS EC2 (t2.micro) dan berjalan stabil. Penelitian ini membuktikan bahwa kombinasi Factory Pattern dan Chain of Responsibility menghasilkan workflow approval yang terstruktur, fleksibel, dan mudah dipelihara.

Kata Kunci: Factory Pattern, Chain of Responsibility, workflow approval, pengadaan berjenjang, audit trail, Spring Boot.

BAB 1 – PENDAHULUAN

1.1 Latar Belakang

Dalam rekayasa perangkat lunak, design pattern memberikan solusi umum untuk masalah yang berulang. Untuk sistem approval yang aturannya bergantung pada nilai nominal, pendekatan prosedural akan sulit dipelihara. Factory Pattern dan Chain of Responsibility dipilih karena mampu memisahkan pembuatan workflow dari logika bisnis, serta memberikan fleksibilitas jika aturan berubah di masa depan.

Berdasarkan observasi awal terhadap 3 perusahaan skala menengah, ditemukan bahwa 100% responden internal mengeluhkan ketidakjelasan jenjang approval dan kurangnya dokumentasi riwayat persetujuan. Oleh karena itu, dibangun sistem approval berjenjang dengan ketentuan:

  • ≤ 50 juta : cukup disetujui oleh Manager → status APPROVED.
  • 51 – 250 juta : Manager → Kepala Divisi → APPROVED.
  • > 250 juta (maksimal 1 Milyar) : Manager → Kepala Divisi → Direksi → APPROVED.

1.2 Tujuan Penelitian

  • Menerapkan Factory Pattern untuk membuat workflow approval secara terpusat dan fleksibel.
  • Mengimplementasikan Chain of Responsibility agar setiap level approval diproses secara manual tanpa auto-forward.
  • Menyediakan REST API dan antarmuka web (HTML/CSS/JS) untuk simulasi tiga role.
  • Mendokumentasikan seluruh tahap desain, kode, dan deployment di AWS EC2.

7.3 ProcurementRequest.java (Model/Entity Pengajuan)

package com.example.procurementfactory.model; import java.util.ArrayList; // Untuk membuat list riwayat import java.util.Date; // Untuk mencatat timestamp approval import java.util.List; // Untuk menyimpan daftar riwayat approval import java.util.UUID; // Untuk menghasilkan ID unik (acak) // Kelas yang merepresentasikan satu pengajuan pengadaan public class ProcurementRequest { private String id; // ID unik pengajuan (auto-generated) private String description; // Deskripsi barang/jasa yang diajukan private double amount; // Nilai pengadaan dalam Rupiah private ApprovalStatus status; // Status approval saat ini (enum) private List<ApprovalHistoryEntry> approvalHistory; // Daftar riwayat approval // Konstruktor: dipanggil saat membuat pengajuan baru public ProcurementRequest(String description, double amount) { this.id = UUID.randomUUID().toString().substring(0, 8); // Generate ID 8 karakter unik this.description = description; this.amount = amount; this.status = ApprovalStatus.WAITING_MANAGER; // Status awal: menunggu Manager this.approvalHistory = new ArrayList<>(); // Inisialisasi list riwayat kosong } // ========== GETTER & SETTER ========== public String getId() { return id; } public String getDescription() { return description; } public double getAmount() { return amount; } public ApprovalStatus getStatus() { return status; } public void setStatus(ApprovalStatus status) { this.status = status; } public List<ApprovalHistoryEntry> getApprovalHistory() { return approvalHistory; } // Method untuk menambahkan riwayat approval baru public void addHistory(String role, String remark) { approvalHistory.add(new ApprovalHistoryEntry(role, remark, new Date())); } // LOGIKA INTI ATURAN BISNIS public ApprovalStatus nextStatusAfter(ApprovalStatus current) { switch (current) { case WAITING_MANAGER: if (amount <= 50_000_000) return ApprovalStatus.APPROVED; else return ApprovalStatus.WAITING_DIVISION_HEAD; case WAITING_DIVISION_HEAD: if (amount <= 250_000_000) return ApprovalStatus.APPROVED; else return ApprovalStatus.WAITING_DIRECTOR; case WAITING_DIRECTOR: return ApprovalStatus.APPROVED; default: return ApprovalStatus.APPROVED; } } // Inner Class: Entri riwayat approval public static class ApprovalHistoryEntry { private String role; private String remark; private Date timestamp; public ApprovalHistoryEntry(String role, String remark, Date timestamp) { this.role = role; this.remark = remark; this.timestamp = timestamp; } public String getRole() { return role; } public String getRemark() { return remark; } public Date getTimestamp() { return timestamp; } } }

7.5 ManagerHandler.java (Handler Level Manager)

package com.example.procurementfactory.handler; import com.example.procurementfactory.model.ProcurementRequest; import com.example.procurementfactory.model.ApprovalStatus; public class ManagerHandler implements ApprovalHandler { private ApprovalHandler next; @Override public void setNext(ApprovalHandler next) { this.next = next; } @Override public void handle(ProcurementRequest request) { if (request.getStatus() == ApprovalStatus.WAITING_MANAGER) { request.addHistory("MANAGER", "Manager menyetujui (nominal: " + request.getAmount() + ")"); request.setStatus(request.nextStatusAfter(ApprovalStatus.WAITING_MANAGER)); } else if (next != null) { next.handle(request); } } }

7.6 DivisionHeadHandler.java

package com.example.procurementfactory.handler; import com.example.procurementfactory.model.ProcurementRequest; import com.example.procurementfactory.model.ApprovalStatus; public class DivisionHeadHandler implements ApprovalHandler { private ApprovalHandler next; @Override public void setNext(ApprovalHandler next) { this.next = next; } @Override public void handle(ProcurementRequest request) { if (request.getStatus() == ApprovalStatus.WAITING_DIVISION_HEAD) { request.addHistory("DIVISION_HEAD", "Kepala Divisi menyetujui (nominal: " + request.getAmount() + ")"); request.setStatus(request.nextStatusAfter(ApprovalStatus.WAITING_DIVISION_HEAD)); } else if (next != null) { next.handle(request); } } }

7.7 DirectorHandler.java

package com.example.procurementfactory.handler; import com.example.procurementfactory.model.ProcurementRequest; import com.example.procurementfactory.model.ApprovalStatus; public class DirectorHandler implements ApprovalHandler { private ApprovalHandler next; @Override public void setNext(ApprovalHandler next) { this.next = next; } @Override public void handle(ProcurementRequest request) { if (request.getStatus() == ApprovalStatus.WAITING_DIRECTOR) { request.addHistory("DIRECTOR", "Direksi menyetujui (nominal: " + request.getAmount() + ")"); request.setStatus(request.nextStatusAfter(ApprovalStatus.WAITING_DIRECTOR)); } else if (next != null) { next.handle(request); } } }

7.8 ApprovalHandlerFactory.java (Factory Pattern)

package com.example.procurementfactory.handler; public class ApprovalHandlerFactory { public static ApprovalHandler createWorkflow() { ManagerHandler manager = new ManagerHandler(); DivisionHeadHandler division = new DivisionHeadHandler(); DirectorHandler director = new DirectorHandler(); manager.setNext(division); division.setNext(director); return manager; } }

7.9 ApprovalService.java (Service Layer)

package com.example.procurementfactory.service; import com.example.procurementfactory.model.ProcurementRequest; import com.example.procurementfactory.model.ApprovalStatus; import com.example.procurementfactory.handler.ApprovalHandler; import com.example.procurementfactory.handler.ApprovalHandlerFactory; import org.springframework.stereotype.Service; import java.util.*; import java.util.stream.Collectors; @Service public class ApprovalService { private final Map<String, ProcurementRequest> requests = new HashMap<>(); private final ApprovalHandler workflowEngine; public ApprovalService() { this.workflowEngine = ApprovalHandlerFactory.createWorkflow(); submitRequest("Laptop kantor", 45_000_000); submitRequest("Software ERP", 150_000_000); submitRequest("Renovasi gedung", 800_000_000); } public ProcurementRequest submitRequest(String description, double amount) { if (amount > 1_000_000_000) amount = 1_000_000_000; ProcurementRequest req = new ProcurementRequest(description, amount); requests.put(req.getId(), req); return req; } public List<ProcurementRequest> getPendingRequestsByRole(String role) { ApprovalStatus requiredStatus = mapRoleToStatus(role); if (requiredStatus == null) return Collections.emptyList(); return requests.values().stream() .filter(r -> r.getStatus() == requiredStatus) .collect(Collectors.toList()); } private ApprovalStatus mapRoleToStatus(String role) { switch (role.toUpperCase()) { case "MANAGER": return ApprovalStatus.WAITING_MANAGER; case "DIVISION_HEAD": return ApprovalStatus.WAITING_DIVISION_HEAD; case "DIRECTOR": return ApprovalStatus.WAITING_DIRECTOR; default: return null; } } public boolean approveRequest(String requestId, String role) { ProcurementRequest req = requests.get(requestId); if (req == null) return false; ApprovalStatus expected = mapRoleToStatus(role); if (req.getStatus() != expected) return false; workflowEngine.handle(req); return true; } public ProcurementRequest getRequest(String id) { return requests.get(id); } public List<ProcurementRequest> getAllRequests() { return new ArrayList<>(requests.values()); } }

7.10 ApprovalController.java (Controller/REST API)

package com.example.procurementfactory.controller; import com.example.procurementfactory.model.ProcurementRequest; import com.example.procurementfactory.service.ApprovalService; import org.springframework.http.ResponseEntity; import org.springframework.stereotype.Controller; import org.springframework.web.bind.annotation.*; import java.util.List; import java.util.Map; @Controller public class ApprovalController { private final ApprovalService service; public ApprovalController(ApprovalService service) { this.service = service; } @GetMapping("/") public String index() { return "forward:/index.html"; } @GetMapping("/api/requests/{role}") @ResponseBody public List<ProcurementRequest> getRequests(@PathVariable String role) { return service.getPendingRequestsByRole(role); } @PostMapping("/api/approve/{id}/{role}") @ResponseBody public ResponseEntity<?> approve(@PathVariable String id, @PathVariable String role) { boolean success = service.approveRequest(id, role); if (success) { ProcurementRequest req = service.getRequest(id); return ResponseEntity.ok(Map.of("success", true, "newStatus", req.getStatus().toString())); } else { return ResponseEntity.badRequest().body(Map.of("success", false)); } } @PostMapping("/api/submit") @ResponseBody public ProcurementRequest submit(@RequestBody Map<String, Object> payload) { String desc = (String) payload.get("description"); double amount = Double.parseDouble(payload.get("amount").toString()); return service.submitRequest(desc, amount); } @GetMapping("/api/all") @ResponseBody public List<ProcurementRequest> allRequests() { return service.getAllRequests(); } @GetMapping("/api/history/{id}") @ResponseBody public List<ProcurementRequest.ApprovalHistoryEntry> getHistory(@PathVariable String id) { ProcurementRequest req = service.getRequest(id); return req != null ? req.getApprovalHistory() : List.of(); } }
