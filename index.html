ABSTRAK

Implementasi Factory Pattern & Chain of Responsibility pada Workflow Persetujuan Pengadaan Barang/Jasa. Penelitian ini bertujuan membangun sistem approval pengadaan berjenjang yang adaptif terhadap nilai transaksi menggunakan pendekatan design pattern. Berdasarkan studi awal pada 3 perusahaan skala menengah, ditemukan keluhan utama berupa ketidakjelasan jenjang approval (100% responden) dan minimnya riwayat persetujuan (100%). Sistem yang diusulkan menerapkan Factory Pattern guna membuat workflow handler secara terpusat serta Chain of Responsibility untuk memproses persetujuan sesuai nominal (≤50 jt, 51–250 jt, >250 jt).

Implementasi menggunakan Spring Boot (Java 11) dengan tiga role: Manager, Kepala Divisi, dan Direksi. Setiap approval mencatat riwayat lengkap (role, remark, timestamp). REST API disediakan untuk antarmuka web (HTML/CSS/JS). Hasil user testing terhadap 3 responden menunjukkan tingkat keberhasilan 88.9% setelah iterasi perbaikan pada tombol approve dan tampilan status. Sistem telah dideploy di AWS EC2 (t2.micro) dan berjalan stabil. Penelitian ini membuktikan bahwa kombinasi Factory Pattern dan Chain of Responsibility menghasilkan workflow approval yang terstruktur, fleksibel, dan mudah dipelihara.

Kata Kunci: Factory Pattern, Chain of Responsibility, workflow approval, pengadaan berjenjang, audit trail, Spring Boot, AWS EC2.

BAB 1 – PENDAHULUAN

1.1 Latar Belakang

Dalam rekayasa perangkat lunak, design pattern memberikan solusi umum untuk masalah yang berulang. Untuk sistem approval yang aturannya bergantung pada nilai nominal, pendekatan prosedural akan sulit dipelihara. Factory Pattern dan Chain of Responsibility dipilih karena mampu memisahkan pembuatan workflow dari logika bisnis, serta memberikan fleksibilitas jika aturan berubah di masa depan.

Berdasarkan observasi awal terhadap 3 perusahaan skala menengah, ditemukan bahwa 100% responden internal mengeluhkan ketidakjelasan jenjang approval dan kurangnya dokumentasi riwayat persetujuan. Oleh karena itu, dibangun sistem approval berjenjang dengan ketentuan:

  • ≤ 50 juta : cukup disetujui oleh Manager → status APPROVED.
  • 51 – 250 juta : Manager → Kepala Divisi → APPROVED.
  • > 250 juta (maksimal 1 Milyar) : Manager → Kepala Divisi → Direksi → APPROVED.

Sistem mencatat history approval (role, remark, timestamp) sebagai audit trail. Antarmuka web disediakan agar ketiga role dapat mensimulasikan approval secara manual.

1.2 Tujuan Penelitian

  • Menerapkan Factory Pattern untuk membuat workflow approval secara terpusat dan fleksibel.
  • Mengimplementasikan Chain of Responsibility agar setiap level approval dapat diproses secara manual tanpa auto-forward.
  • Menyediakan REST API dan antarmuka web (HTML/CSS/JS) untuk simulasi tiga role.
  • Mendokumentasikan seluruh tahap desain, kode, dan deployment di AWS EC2.

1.3 Objek Riset

  • Aplikasi: Sistem Approval Pengadaan Berjenjang (Java Spring Boot)
  • Fitur yang dianalisis: workflow approval eksisting (belum terstruktur)
  • Fitur yang diusulkan: workflow engine berbasis Factory Pattern + Chain of Responsibility

BAB 5 – PENJELASAN METODE PROGRAM JAVA

Setiap class memiliki tanggung jawab spesifik dalam sistem approval berjenjang ini:

  • ProcurementRequest.java: Model data yang menyimpan informasi pengajuan dan riwayat approval. Method nextStatusAfter() mengimplementasikan aturan bisnis penentuan status berikutnya berdasarkan nominal.
  • ApprovalStatus.java: Enum yang mendefinisikan 4 status (WAITING_MANAGER, WAITING_DIVISION_HEAD, WAITING_DIRECTOR, APPROVED).
  • ApprovalHandler (Interface): Kontrak untuk semua handler dalam Chain of Responsibility dengan method handle() dan setNext().
  • ManagerHandler, DivisionHeadHandler, DirectorHandler: Implementasi konkret Chain of Responsibility. Setiap handler hanya memproses jika status sesuai, lalu memperbarui status. Tidak ada auto-forward untuk memastikan approval manual per level.
  • ApprovalHandlerFactory: Implementasi Factory Pattern yang membuat dan merangkai ketiga handler menjadi workflow engine.
  • ApprovalService: Service layer yang menyimpan data (Map in-memory) dan mengatur logika bisnis seperti submit request, filter pending request, dan menjalankan workflow saat approve.
  • ApprovalController: REST Controller yang menyediakan endpoint untuk frontend (GET/POST).
  • ProcurementApplication: Main class Spring Boot dengan annotation @SpringBootApplication.

7.3 ProcurementRequest.java – Model/Entity Pengajuan

📌 Fungsi: Kelas ini merepresentasikan satu pengajuan pengadaan. Atribut utama: id (unik), description, amount (nominal), status (enum), approvalHistory (daftar riwayat). Method nextStatusAfter() adalah inti aturan bisnis yang menentukan status berikutnya berdasarkan nominal. Method addHistory() mencatat riwayat approval dengan timestamp otomatis.
package com.example.procurementfactory.model; import java.util.ArrayList; // Untuk membuat list riwayat import java.util.Date; // Untuk mencatat timestamp approval import java.util.List; // Untuk menyimpan daftar riwayat approval import java.util.UUID; // Untuk menghasilkan ID unik (acak) // Kelas yang merepresentasikan satu pengajuan pengadaan public class ProcurementRequest { private String id; // ID unik pengajuan (auto-generated) private String description; // Deskripsi barang/jasa yang diajukan private double amount; // Nilai pengadaan dalam Rupiah private ApprovalStatus status; // Status approval saat ini (enum) private List<ApprovalHistoryEntry> approvalHistory; // Daftar riwayat approval // Konstruktor: dipanggil saat membuat pengajuan baru public ProcurementRequest(String description, double amount) { this.id = UUID.randomUUID().toString().substring(0, 8); // Generate ID 8 karakter unik this.description = description; this.amount = amount; this.status = ApprovalStatus.WAITING_MANAGER; // Status awal: menunggu Manager this.approvalHistory = new ArrayList<>(); // Inisialisasi list riwayat kosong } // GETTER (mengambil nilai atribut) public String getId() { return id; } public String getDescription() { return description; } public double getAmount() { return amount; } public ApprovalStatus getStatus() { return status; } public void setStatus(ApprovalStatus status) { this.status = status; } // SETTER public List<ApprovalHistoryEntry> getApprovalHistory() { return approvalHistory; } // Method untuk menambahkan riwayat approval baru public void addHistory(String role, String remark) { approvalHistory.add(new ApprovalHistoryEntry(role, remark, new Date())); } // LOGIKA INTI ATURAN BISNIS: menentukan status berikutnya berdasarkan nominal public ApprovalStatus nextStatusAfter(ApprovalStatus current) { switch (current) { case WAITING_MANAGER: // Jika baru dari Manager if (amount <= 50_000_000) return ApprovalStatus.APPROVED; // ≤50jt langsung APPROVED else return ApprovalStatus.WAITING_DIVISION_HEAD; // >50jt lanjut ke Kepala Divisi case WAITING_DIVISION_HEAD: // Jika dari Kepala Divisi if (amount <= 250_000_000) return ApprovalStatus.APPROVED; // ≤250jt langsung APPROVED else return ApprovalStatus.WAITING_DIRECTOR; // >250jt lanjut ke Direksi case WAITING_DIRECTOR: // Jika dari Direksi return ApprovalStatus.APPROVED; // Langsung APPROVED (selesai) default: return ApprovalStatus.APPROVED; } } // Inner Class: Entri riwayat approval (menyimpan detail setiap persetujuan) public static class ApprovalHistoryEntry { private String role; // Role yang melakukan approval private String remark; // Keterangan/catatan approval private Date timestamp; // Waktu approval dilakukan public ApprovalHistoryEntry(String role, String remark, Date timestamp) { this.role = role; this.remark = remark; this.timestamp = timestamp; } public String getRole() { return role; } public String getRemark() { return remark; } public Date getTimestamp() { return timestamp; } } }

7.4 ApprovalHandler.java – Interface Handler

📌 Fungsi: Interface ini adalah kontrak untuk semua handler dalam Chain of Responsibility Pattern. Method handle() digunakan untuk memproses request approval, method setNext() untuk menyambungkan ke handler berikutnya. Setiap class yang mengimplementasikan interface ini akan menjadi satu mata rantai dalam workflow approval.
package com.example.procurementfactory.handler; import com.example.procurementfactory.model.ProcurementRequest; // Interface untuk Chain of Responsibility Pattern // Setiap handler approval harus mengimplementasikan interface ini public interface ApprovalHandler { void handle(ProcurementRequest request); // Method untuk memproses approval request void setNext(ApprovalHandler next); // Method untuk menyetel handler berikutnya dalam rantai }

7.5 ManagerHandler.java – Handler Level Manager

📌 Fungsi: Handler untuk level Manager. Hanya akan memproses request dengan status WAITING_MANAGER. Saat approve: mencatat history, lalu menentukan status berikutnya melalui nextStatusAfter(). Penting: Tidak otomatis memanggil handler berikutnya – ini memastikan approval dilakukan secara manual per level oleh user dengan role yang sesuai.
package com.example.procurementfactory.handler; import com.example.procurementfactory.model.ProcurementRequest; import com.example.procurementfactory.model.ApprovalStatus; public class ManagerHandler implements ApprovalHandler { private ApprovalHandler next; // Referensi ke handler berikutnya (Kepala Divisi) @Override public void setNext(ApprovalHandler next) { this.next = next; } @Override public void handle(ProcurementRequest request) { if (request.getStatus() == ApprovalStatus.WAITING_MANAGER) { request.addHistory("MANAGER", "Manager menyetujui (nominal: " + request.getAmount() + ")"); request.setStatus(request.nextStatusAfter(ApprovalStatus.WAITING_MANAGER)); // TIDAK LANJUT OTOMATIS → approval manual per level } else if (next != null) { next.handle(request); // Jika status tidak sesuai, lanjut ke handler berikutnya } } }

7.6 DivisionHeadHandler.java – Handler Level Kepala Divisi

📌 Fungsi: Handler untuk level Kepala Divisi. Memproses request dengan status WAITING_DIVISION_HEAD. Setelah approve, status akan menjadi APPROVED (jika nominal ≤250jt) atau WAITING_DIRECTOR (jika nominal >250jt). History dicatat setiap kali approval dilakukan.
package com.example.procurementfactory.handler; import com.example.procurementfactory.model.ProcurementRequest; import com.example.procurementfactory.model.ApprovalStatus; public class DivisionHeadHandler implements ApprovalHandler { private ApprovalHandler next; @Override public void setNext(ApprovalHandler next) { this.next = next; } @Override public void handle(ProcurementRequest request) { if (request.getStatus() == ApprovalStatus.WAITING_DIVISION_HEAD) { request.addHistory("DIVISION_HEAD", "Kepala Divisi menyetujui (nominal: " + request.getAmount() + ")"); request.setStatus(request.nextStatusAfter(ApprovalStatus.WAITING_DIVISION_HEAD)); } else if (next != null) { next.handle(request); } } }

7.7 DirectorHandler.java – Handler Level Direksi (Tertinggi)

📌 Fungsi: Handler untuk level Direksi – level tertinggi dalam rantai approval. Memproses request dengan status WAITING_DIRECTOR. Setelah approve, status akan menjadi APPROVED (selesai). Tidak ada handler setelah ini karena ini adalah level terakhir.
package com.example.procurementfactory.handler; import com.example.procurementfactory.model.ProcurementRequest; import com.example.procurementfactory.model.ApprovalStatus; public class DirectorHandler implements ApprovalHandler { private ApprovalHandler next; @Override public void setNext(ApprovalHandler next) { this.next = next; } @Override public void handle(ProcurementRequest request) { if (request.getStatus() == ApprovalStatus.WAITING_DIRECTOR) { request.addHistory("DIRECTOR", "Direksi menyetujui (nominal: " + request.getAmount() + ")"); request.setStatus(request.nextStatusAfter(ApprovalStatus.WAITING_DIRECTOR)); } else if (next != null) { next.handle(request); } } }

7.8 ApprovalHandlerFactory.java – Factory Pattern

📌 Fungsi: Inilah inti dari Factory Pattern. Method static createWorkflow() membuat dan merangkai ketiga handler secara berurutan: Manager → DivisionHead → Director. Handler pertama (Manager) dikembalikan sebagai entry point workflow. Keuntungan: perubahan urutan handler cukup dilakukan di sini tanpa mengubah class lain (Open/Closed Principle).
package com.example.procurementfactory.handler; // Factory Pattern: Kelas yang bertanggung jawab membuat dan merangkai handler-chain public class ApprovalHandlerFactory { public static ApprovalHandler createWorkflow() { ManagerHandler m = new ManagerHandler(); DivisionHeadHandler d = new DivisionHeadHandler(); DirectorHandler dir = new DirectorHandler(); m.setNext(d); d.setNext(dir); // Merangkai: Manager → Division → Director return m; // Mengembalikan handler pertama sebagai entry point } }

7.9 ApprovalService.java – Service Layer

📌 Fungsi: Kelas ini adalah pusat logika bisnis. requests (Map) adalah simulasi database in-memory. workflowEngine adalah hasil dari Factory Pattern. Method submitRequest() membuat pengajuan baru. getPendingRequestsByRole() memfilter request berdasarkan role. approveRequest() memanggil workflowEngine.handle() untuk menjalankan Chain of Responsibility.
package com.example.procurementfactory.service; import com.example.procurementfactory.model.ProcurementRequest; import com.example.procurementfactory.model.ApprovalStatus; import com.example.procurementfactory.handler.ApprovalHandler; import com.example.procurementfactory.handler.ApprovalHandlerFactory; import org.springframework.stereotype.Service; import java.util.*; import java.util.stream.Collectors; @Service public class ApprovalService { private final Map<String, ProcurementRequest> requests = new HashMap<>(); // Simulasi DB in-memory private final ApprovalHandler workflowEngine = ApprovalHandlerFactory.createWorkflow(); public ApprovalService() { submitRequest("Laptop kantor", 45_000_000); submitRequest("Software ERP", 150_000_000); submitRequest("Renovasi gedung", 800_000_000); } public ProcurementRequest submitRequest(String description, double amount) { if (amount > 1_000_000_000) amount = 1_000_000_000; ProcurementRequest req = new ProcurementRequest(description, amount); requests.put(req.getId(), req); return req; } public List<ProcurementRequest> getPendingRequestsByRole(String role) { ApprovalStatus requiredStatus = mapRoleToStatus(role); if (requiredStatus == null) return Collections.emptyList(); return requests.values().stream() .filter(r -> r.getStatus() == requiredStatus) .collect(Collectors.toList()); } private ApprovalStatus mapRoleToStatus(String role) { switch (role.toUpperCase()) { case "MANAGER": return ApprovalStatus.WAITING_MANAGER; case "DIVISION_HEAD": return ApprovalStatus.WAITING_DIVISION_HEAD; case "DIRECTOR": return ApprovalStatus.WAITING_DIRECTOR; default: return null; } } public boolean approveRequest(String requestId, String role) { ProcurementRequest req = requests.get(requestId); if (req == null) return false; if (req.getStatus() != mapRoleToStatus(role)) return false; workflowEngine.handle(req); // Menjalankan workflow engine (Chain of Responsibility) return true; } public ProcurementRequest getRequest(String id) { return requests.get(id); } public List<ProcurementRequest> getAllRequests() { return new ArrayList<>(requests.values()); } }

7.10 ApprovalController.java – REST Controller

📌 Fungsi: Menyediakan endpoint REST untuk komunikasi frontend-backend. @GetMapping("/") mengarahkan ke index.html. /api/requests/{role} mengembalikan daftar request pending. /api/approve/{id}/{role} memanggil service.approveRequest(). /api/submit menerima data JSON untuk membuat pengajuan baru. /api/history/{id} mengembalikan riwayat approval.
package com.example.procurementfactory.controller; import com.example.procurementfactory.model.ProcurementRequest; import com.example.procurementfactory.service.ApprovalService; import org.springframework.http.ResponseEntity; import org.springframework.stereotype.Controller; import org.springframework.web.bind.annotation.*; import java.util.List; import java.util.Map; @Controller public class ApprovalController { private final ApprovalService service; public ApprovalController(ApprovalService service) { this.service = service; } @GetMapping("/") public String index() { return "forward:/index.html"; } @GetMapping("/api/requests/{role}") @ResponseBody public List<ProcurementRequest> getRequests(@PathVariable String role) { return service.getPendingRequestsByRole(role); } @PostMapping("/api/approve/{id}/{role}") @ResponseBody public ResponseEntity<?> approve(@PathVariable String id, @PathVariable String role) { boolean success = service.approveRequest(id, role); if (success) { ProcurementRequest req = service.getRequest(id); return ResponseEntity.ok(Map.of("success", true, "newStatus", req.getStatus().toString())); } else { return ResponseEntity.badRequest().body(Map.of("success", false)); } } @PostMapping("/api/submit") @ResponseBody public ProcurementRequest submit(@RequestBody Map<String, Object> payload) { String desc = (String) payload.get("description"); double amount = Double.parseDouble(payload.get("amount").toString()); return service.submitRequest(desc, amount); } @GetMapping("/api/all") @ResponseBody public List<ProcurementRequest> allRequests() { return service.getAllRequests(); } @GetMapping("/api/history/{id}") @ResponseBody public List<ProcurementRequest.ApprovalHistoryEntry> getHistory(@PathVariable String id) { ProcurementRequest req = service.getRequest(id); return req != null ? req.getApprovalHistory() : List.of(); } }

7.11 ProcurementApplication.java – Main Class

📌 Fungsi: Kelas utama dengan annotation @SpringBootApplication. Method main() menjalankan SpringApplication.run() yang akan memulai embedded server (Tomcat), melakukan auto-configuration, dan memindai semua component (@Controller, @Service). Ini adalah pintu masuk aplikasi saat dijalankan dengan java -jar.
package com.example.procurementfactory; import org.springframework.boot.SpringApplication; import org.springframework.boot.autoconfigure.SpringBootApplication; @SpringBootApplication public class ProcurementApplication { public static void main(String[] args) { SpringApplication.run(ProcurementApplication.class, args); // Entry point aplikasi Spring Boot } }

BAB 8 – DEPLOYMENT & OPERASI (AWS EC2)

8.1 Persiapan EC2 Instance

📌 Fungsi: EC2 (Elastic Compute Cloud) adalah layanan AWS untuk menyewa server virtual. Instance t2.micro dipilih karena termasuk dalam free tier. Security Group berfungsi sebagai firewall virtual untuk mengontrol traffic masuk.
  • Buat instance EC2 t2.micro (Amazon Linux 2023 atau Ubuntu 22.04)
  • Pada Security Group, buka port 8081 (aplikasi) dan 22 (SSH)
  • Catat Public IP instance

8.2 Instal Java 11 di EC2

📌 Fungsi: Aplikasi Spring Boot membutuhkan Java Runtime Environment (JRE) versi 11 untuk menjalankan file JAR. Perintah di bawah menginstal OpenJDK 11, yaitu implementasi open-source dari Java SE.
# Untuk Ubuntu/Debian sudo apt update && sudo apt install openjdk-11-jdk -y # Untuk Amazon Linux sudo dnf install java-11-amazon-corretto-devel # Verifikasi instalasi java -version # Output harus menampilkan versi 11.x.x

8.3 Build & Jalankan Aplikasi

📌 Fungsi: Maven digunakan untuk mengcompile dan mengemas aplikasi menjadi file JAR. Perintah nohup dan & digunakan untuk menjalankan aplikasi di background agar tetap berjalan meskipun sesi SSH ditutup.
# Build dengan Maven (hasilkan file JAR) mvn clean package # Jalankan foreground (untuk testing langsung) java -jar target/procurement-approval-factory-1.0.jar --server.port=8081 # Jalankan background (tahan close terminal) - REKOMENDASI nohup java -jar target/procurement-approval-factory-1.0.jar --server.port=8081 > app.log 2>&1 & # Alternatif dengan disown java -jar target/procurement-approval-factory-1.0.jar --server.port=8081 & disown

8.4 Akses Halaman Web

📌 Fungsi: Setelah aplikasi berjalan di server, kita dapat mengakses antarmuka web dari browser melalui Public IP EC2 pada port 8081. Pastikan Security Group sudah mengizinkan inbound traffic port 8081.
http://13.229.109.30:8081/ # Ganti dengan Public IP EC2 Anda

Contoh output yang diharapkan: Halaman web dengan form input pengajuan, dropdown role, dan tabel riwayat approval.
